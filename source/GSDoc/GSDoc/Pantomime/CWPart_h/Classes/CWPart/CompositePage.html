CWPart

Discussion

This class defines what Internet messages are composed of. An Internet message is composed of various parts which can be text parts, images, PDF documents and so on. Even a message is a part, with additional headers (like "From:", "Subject:" and so on). CWMessage is a superclass of CWPart.

- (NSString *) contentType;
Return Value

The value. If none was set, nil will be returned.

Discussion

This method is used to get the value of the receiver's "Content-Type" header. If none is specified, nil will be returned. It's up to the callee to use whatever charset it wants. Section 5.2 or RFC2045 recommends to use "text/plain" in this case.

dataValue

- (NSData *) dataValue;
Return Value

The encoded CWPart instance, as a NSData instance.

Discussion

This method is used to encoded the receiver's using the MIME standard before it is ready for submisssion (using a Transport method) or being saved on disk. The CWMessage class overwrites this method in order to add Message-specific headers such as "From", "Subject" and so on.

initWithData:

- (id) initWithData: (NSData *) theData;
Parameters
theData

The bytes to use.

Return Value

A CWPart instance, nil on error.

Discussion

This method is used to initialize the CWPart instance with the specified data. It splits the header part with the content part and calls -setHeadersFromData: with the header part and CWMIMEUtility: +setContentFromRawSource: inPart: with the content part. This method will recursively go in sub-parts (if needed) and initilize them also.
