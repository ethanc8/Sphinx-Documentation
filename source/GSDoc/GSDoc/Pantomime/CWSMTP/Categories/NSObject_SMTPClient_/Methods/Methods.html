recipientIdentificationCompleted:

- ( void ) recipientIdentificationCompleted:
        (NSNotification *) theNotification;

This method is automatically invoked on the delegate after the SMTP RCPT TO command has completed. The userInfo's part of the notification contains the recipients (CWInternetAddress instances) that were sucessfully sent. The key to obtain it is "Recipients". A PantomimeRecipientIdentificationCompleted notification is also posted.

Parameter Descriptions
theNotification
The notification holding the information.

recipientIdentificationFailed:

- ( void ) recipientIdentificationFailed:
        (NSNotification *) theNotification;

This method is automatically invoked on the delegate after the SMTP RCPT TO command has failed. The userInfo's part of the notification contains the recipient (CWInternetAddress instance) that was not sucessfully sent. The key to obtain it is "Recipient". A PantomimeRecipientIdentificationFailed notification is also posted.

Parameter Descriptions
theNotification
The notification holding the information.

transactionResetFailed:

- ( void ) transactionResetFailed:
        (NSNotification *) theNotification;

This method is automatically invoked on the delegate after the SMTP RSET command has completed and failed. This should never really happen but this method could be invoked, if the SMTP server isnt RFC-strict. A PantomimeTransactionResetFailed notification is also posted.

Parameter Descriptions
theNotification
The notification holding the information.
