prefetch

- (void) prefetch;
Discussion

This method is used to cache a certain amount of information from the POP3 server, for all messages. It caches the number of messages, their size and UID. On completion, it posts a PantomimeFolderPrefetchCompleted notification (and calls -folderPrefetchCompleted: on the delegate, if any). This method is also fully asynchronous.

prefetchMessageAtIndex: numberOfLines:

- (void) prefetchMessageAtIndex: (unsigned int) theIndex 
        numberOfLines: (unsigned int) theNumberOfLines;
Parameters
theIndex

The index of the message, which is 1-based in POP3.

theNumberOfLines

The number of lines to fetch. Specifying 0 will only fetch the headers.

Discussion

This method is used to fetch a certain amount of lines of the message, at the specified index. On completion, this method posts the PantomimeMessagePrefetchCompleted (and calls -messagePrefetchCompleted: on the delegate, if any). This method is fully asynchronous. Since this method can be used to partially fetch the message, it is the responsability of the caller to verify if the message's data is fully loaded. One can verify the lenght of the message's data and compare it to the message's size.

retainPeriod

- (unsigned int) retainPeriod;
Return Value

The retain period. A value of 0 means that the messages will always be retained.

Discussion

This method is used to obtain the retain period for the messages, if left on the server.

setLeaveOnServer:

- (void) setLeaveOnServer: (BOOL) theBOOL;
Parameters
theBOOL

YES if we want to leave them on the server after a fetch. If NO, the messages will be automatically deleted after being fetched.

Discussion

This method is used to set the flag to leave or not the messages on the POP3 server after fetching them.

setRetainPeriod:

- (void) setRetainPeriod: (unsigned int) theRetainPeriod;
Parameters
theRetainPeriod

The retain period to use, in number of days.

Discussion

This method is used to set the retain period (in days) before the messages are deleted from the server. In order for this to work, the receiver must have a cache (instance of CWPOP3CacheManager).
