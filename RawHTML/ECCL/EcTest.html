EcTest functions

EcTestConnect

id<EcTest> EcTestConnect(NSString* name, NSString* host, NSTimeInterval timeout);
This function obtains a Distributed Objects proxy to the EcProcess instance controlling the server with the specified name and host.
A nil or empty string for the host is taken to mean the local host, while an asterisk denotes any host on the local network.
The timeout is a time limit on how long it may take to get the connection (a value less than or equal to zero will cause the function to keep on trying indefinitely).
If an immediate connection is not possible, the function will attempt to contact the Command server on the specified host and ask it to launch the process before trying to connect again.

EcTestGetConfig

id EcTestGetConfig(id<EcTest> process, NSString* key);
This function gets process configuration for the specified key and deserialises it to a property list object (returned) or nil if no value is configured for the specified key .

EcTestSetConfig

void EcTestSetConfig(id<EcTest> process, NSString* key, id value);
This function sets process configuration by serialising the property list value and passing the resulting data to the remote process.
If the value is nil then the configuration for the remote process reverts to its default setting.

EcTestShutdown

BOOL EcTestShutdown(id<EcTest> process, NSTimeInterval timeout);
This function shuts down the remote process, gracefully if possible, but then with a kill if graceful shutdown fails.
Returns YES on graceful shutdown, NO on timeout (or if the connection had already been invalidated).

EcTestShutdownByName

BOOL EcTestShutdownByName(NSString* name, NSString* host, NSTimeInterval timeout);
This function shuts down the remote process, gracefully if possible, but then with a kill if graceful shutdown fails.
Returns YES on graceful shutdown (or if the process was not running), NO on timeout.
