NSUserDefaults(EcUserDefaults)

Declared in:
EcUserDefaults.h
This category simply provides an easy way to work with user defaults where you want all keys to share a common prefix.

prefixedDefaults 

+ (NSUserDefaults*) prefixedDefaults;
Returns the latest prefixed version of the shared user defaults, or nil if none has been set up.

setDefaultLifetime: 

+ (void) setDefaultLifetime: (NSTimeInterval)t;
Sets the default lifetime for command values set using the -setCommand:forKey: method.

userDefaultsWithPrefix: 

+ (NSUserDefaults*) userDefaultsWithPrefix: (NSString*)aPrefix;
Returns a proxy to the shared user defaults instance, which will allow aPrefix at the start of every key.
If aPrefix is nil, the string given by the EcUserDefaultsPrefix user default is used.
The proxy will read defaults using the unprefixed key if no value is found for the prefixed key.

commandExpiries 

- (NSDictionary*) commandExpiries;
Returns a dictionary listing all the command override keys for which values are currently set. The values in the dictionary are the timestamps after which those values may be purged.

commandObjectForKey: 

- (id) commandObjectForKey: (NSString*)aKey;
Returns the last value set for the specified key using the -setCommand:forKey: method. This returns nil if no value is currently set.

configuration 

- (NSDictionary*) configuration;
Returns the current configuration settings dictionary (as set using the -setConfiguration: method).

defaultsPrefix 

- (NSString*) defaultsPrefix;
Returns the prefix used by the receiver, or nil if no prefix is in use.

key: 

- (NSString*) key: (NSString*)aKey;
Convenience method to prepend the prefix to the supplied aKey value if it is not already present.

purgeSettings 

- (void) purgeSettings;
Removes all settings whose lifetime has passed. Those settings must previously have been set up using the -setCommand:forKey:lifetime: method.

revertSettings 

- (void) revertSettings;
Removes all settings previously set up using the -setCommand:forKey: method.

setCommand: forKey: 

- (BOOL) setCommand: (id)val forKey: (NSString*)key;
Sets a value to take precedence over others (in the volatile domain reserved for commands issued to the current process by an operator).
Values set using this method will use the default lifetime.
This operates by using the -setCommand:forKey:lifetime: method.

setCommand: forKey: lifetime: 

- (BOOL) setCommand: (id)val forKey: (NSString*)key lifetime: (NSTimeInterval)t;
Sets a value to take precedence over others (in the volatile domain reserved for commands issued to the current process by an operator).
Specifying a non-zero lifetime will adjust the lifetime of an existing setting irresepective of whether the value is changed or not.
Specifying a zero or negative lifetime will remove the value for the setting (as will setting a nil value).
Returns YES if the configuration (actual value set) was changed, NO otherwise (may have changed lifetime of setting).

setConfiguration: 

- (BOOL) setConfiguration: (NSDictionary*)config;
Replaces the system central configuration information for this process with the contents of the dictionary. Values in this dictionary take precedence over other configured values except for those set using the -setCommand:forKey: method.
Returns YES if the configuration changed, NO otherwise.
